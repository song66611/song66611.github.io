用GitHub和hexo搭建个人博客网站

2022-07-29

本次踩雷很多~~~~
第一次使用的方法:
最开始搭建的时候是参考的这篇文章https://blog.csdn.net/wapchief/article/details/54602515，这个是依靠hexo和GitHub来搭建的博客，本来我看着步骤挺简单的，但上手操作后发现里面有很多的坑，博主写的是没有问题，但我下载的git版本和博主里面的不一样，我的是最新的一版，还有GitHub中也有更新。步骤如下：
1.在按照他的操作步骤来操作的话，在下载好hexo和git还有jode后，在cmd中安装hexo，要用npm命令使用npm install hexo-cli -g 命令安装，安装好后再用hexo -v查看安装成功没有。然后在随便一个文件位置创建一个文件夹名字可以随便，例如GitHub，在GitHub文件夹下再新建个blog文件夹以便放置配置文件

2.右键点击blog文件夹，点击git bash here

3.出现 如图这样的框框后进行命令的输入

npm install hexo --save 初始化hexo基础配置文件。初始化时间比较长，不用急，等完成后输入
hexo init 初始化hexo。这个命令和git 仓库命令相似。意思一样都是初始化。
npm install 配置node。然后输入
hexo g 加载hexo基础html、css、js等文件。
在这完成后等于已经在本地创建了一个网页，想查看的话，输入hexo s
这里面有时候会有一些错误，但都是一些小问题可以解决，因为我当时没有截屏，所以不能把我碰到的问题分享出来。
4.输入hexo s 中的地址,一般都是http://localhost:4000/ ，如果出来这样的页面，则上面操作无误

5.设置ssh，可见这篇文章 https://blog.csdn.net/weixin_45377770/article/details/105228938?spm=1001.2101.3001.6650.2&utm_medium=distribute.pc_relevant.none-task-blog-2%7Edefault%7ECTRLIST%7Edefault-2-105228938-blog-54602515.pc_relevant_multi_platform_whitelistv1_exp2&depth_1-utm_source=distribute.pc_relevant.none-task-blog-2%7Edefault%7ECTRLIST%7Edefault-2-105228938-blog-54602515.pc_relevant_multi_platform_whitelistv1_exp2&utm_relevant_index=5 ，
在这里面有几个坑，（1）在GitHub中配置tokens，配置好了要粘贴复制，后面就看不到密匙了。为什么配置token因为GitHub更新后是取消了用户密码的使用，在git命令中输入hexo d 后会叫你输入用户名和密码，这里密码里面要输入token密匙
5.选择主题的安装，在GitHub中注册登录（网络可能不好可以选择翻墙）外加厂库的创建，在GitHub左上角的搜索框中搜hexo theme 会有很多的主题，选一个进入，复制链接

6.回到git命令中输入“ git clone 复制的主题链接 themes/主题名 ” ，在_config.yml文件最下面有个theme：landscape ,把landscape改成你的主题名，保存
7.配置远程仓库名，也是在_config.yml文件中，在最下面把本来的语句改成这样，repo中放的是自己GitHub中的仓库地址，保存
Deployment
Docs: https://hexo.io/docs/deployment.html
deploy:
type: git
repo: https://github.com/”仓库左上角的地址“
branch: master

8.在命令行输入hexo clean && hexo g && hexo d。然后不出意外就可以在仓库中看到文件了。
9.最后，注意因为我们上传的文件在master中没有在main 中所以要在github page中更改，第三步操作要选master。（我就是没有弄这一步，导致下载的主题加载不出来，所以有了方法二）

10.输入网址访问，就是https://加上你仓库的名字，例如https://song66611.github.io
